  <name>Task 1: Property population audit and test scaffold</name>
  <files>scripts/property_audit.py, tests/unit/test_property_audit.py, tests/integration/test_shacl_validation.py</files>
  <action>
Create `scripts/property_audit.py` that:
1. Loads each of the three TTL files (AOPWikiRDF.ttl, AOPWikiRDF-Genes.ttl, AOPWikiRDF-Enriched.ttl) into separate rdflib Graphs
2. For each file, runs a SPARQL query to discover all rdf:type values and their instance counts
3. For each discovered type, runs a SPARQL query to find all properties used on instances of that type, with counts and population percentages (count/total_instances * 100)
4. Classifies each property as "sh:Violation" (>=90% population or core identity: dc:identifier, dc:title, rdf:type) or "sh:Warning" (<90%)
5. Outputs results as both human-readable table to stdout AND JSON to `scripts/audit-results.json`
6. The JSON structure: `{filename: {type_uri: {instances: N, properties: {prop_uri: {count, total, percentage, severity}}}}}`

Also run the audit against the current data files and save the results to `scripts/audit-results.json`. This JSON file is the input for Task 2.

Create test stubs:
- `tests/unit/test_property_audit.py`: Test that audit_file() returns expected structure (dict with type keys, property sub-dicts with count/total/percentage/severity fields). Use a small inline Turtle string as fixture data rather than loading production files.
- `tests/integration/test_shacl_validation.py`: Create stubs with `pytest.skip("shapes not yet created")` for: test_shacl_validates_aop_entities, test_shacl_validates_gene_associations, test_shacl_validates_enriched_xrefs, test_no_violations_on_current_data, test_completes_under_timeout.

Use exact namespace URIs from `src/aopwiki_rdf/rdf/namespaces.py` -- do not guess or approximate.
  </action>
  <verify>
    <automated>python scripts/property_audit.py && python -m pytest tests/unit/test_property_audit.py -x --timeout=30</automated>
  </verify>
  <done>Audit script produces JSON with population percentages for every entity type across all three TTL files. Unit test confirms audit_file() returns correct structure. Integration test stubs exist for SHACL validation.</done>
</task>

<task type="auto">
  <name>Task 2: Define SHACL shapes from audit results</name>
  <files>shapes/aop-shape.ttl, shapes/key-event-shape.ttl, shapes/ker-shape.ttl, shapes/stressor-shape.ttl, shapes/chemical-shape.ttl, shapes/gene-association-shape.ttl, shapes/enriched-xref-shape.ttl, scripts/run_shacl_validation.py, tests/integration/test_shacl_validation.py</files>
  <action>
Read `scripts/audit-results.json` from Task 1. For each entity type discovered in the audit, create a SHACL shape file in the `shapes/` directory:

1. **shapes/aop-shape.ttl** -- sh:targetClass aopo:AdverseOutcomePathway. Include all properties found in audit. Properties with >=90% population get sh:minCount 1 + sh:Violation severity. Properties with <90% get sh:Warning severity (no sh:minCount). Core identity properties (dc:identifier, dc:title, rdf:type) always get sh:Violation regardless of percentage.

2. **shapes/key-event-shape.ttl** -- sh:targetClass aopo:KeyEvent. Same severity logic.

3. **shapes/ker-shape.ttl** -- sh:targetClass for KER type (discover exact URI from audit -- likely aopo:KeyEventRelationship).

4. **shapes/stressor-shape.ttl** -- sh:targetClass for Stressor (likely nci:C54571 per CONTEXT.md).

5. **shapes/chemical-shape.ttl** -- sh:targetClass for Chemical entities (likely cheminf:000000 or cheminf:000446).

6. **shapes/gene-association-shape.ttl** -- sh:targetClass for gene association type from AOPWikiRDF-Genes.ttl (likely edam:data_1025).

7. **shapes/enriched-xref-shape.ttl** -- For enriched cross-reference patterns from AOPWikiRDF-Enriched.ttl. This may use sh:targetSubjectsOf owl:sameAs if entities lack explicit rdf:type.

All shapes must use the EXACT namespace URIs from `src/aopwiki_rdf/rdf/namespaces.py`. Each shape file must be valid Turtle with proper @prefix declarations.

Create `scripts/run_shacl_validation.py` that:
1. Loads all data TTL files (excluding ServiceDescription.ttl) into one rdflib Graph
2. Loads all shapes from `shapes/*.ttl` into a shapes Graph
3. Runs `pyshacl.validate(data, shacl_graph=shapes, inference=None, abort_on_first=False)`
4. Saves full report to `shacl-report.ttl`
5. Saves summary JSON to `shacl-summary.json` with conforms/violations/warnings/status
6. Exits 0 if no sh:Violation results, exits 1 if any sh:Violation results
7. sh:Warning results are logged but do NOT cause failure

Update `tests/integration/test_shacl_validation.py` to remove skips and add real tests:
- test_shacl_validates_aop_entities: Load AOPWikiRDF.ttl + aop-shape.ttl, run pyshacl, assert no violations
- test_no_violations_on_current_data: Run full validation runner, assert exit code 0
- test_completes_under_timeout: Assert validation completes in under 300 seconds (well within 30-min CI limit)

Install pyshacl: `pip install pyshacl` and add to requirements.txt.
  </action>
