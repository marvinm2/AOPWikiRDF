  <action>
Create `prototypes/biobert_ner/requirements.txt`:
```
transformers>=4.0

```

Create `prototypes/biobert_ner/run_ner.py` that:

1. **Extract KE descriptions** from data/AOPWikiRDF.ttl using rdflib SPARQL query. Select 50-100 KEs that have dc:description values. Choose a variety: molecular-level KEs (likely to mention genes), cellular-level, organ-level, and organism-level KEs. Use LIMIT 100 and filter for descriptions longer than 50 characters.

2. **Run BioBERT NER** using `alvaroalon2/biobert_genetic_ner` via transformers pipeline with `aggregation_strategy="simple"`. For each KE description, extract gene entities with confidence scores. Truncate descriptions to 512 characters for model input. Filter results to entities with score > 0.5.

3. **Run regex baseline** for comparison. Import or replicate the core regex matching logic from the gene mapper's three-stage algorithm. For each KE description, find gene matches using the HGNC gene dictionary. This gives the current production baseline.

4. **Compare results** per KE description:
   - Genes found by BioBERT only (potential catches regex misses)
   - Genes found by regex only (potential BioBERT misses)
   - Genes found by both (agreement)
   - For a manually annotated subset of ~20 KE descriptions, calculate true precision and recall for both methods

5. **Output results** to `prototypes/biobert_ner/results/`:
   - `comparison.json`: Per-KE comparison data
   - `summary.json`: Aggregate precision/recall/F1 for both methods
   - `disagreements.json`: Cases where methods disagree (most interesting for analysis)

6. **Runtime measurement**: Time both methods and report seconds per KE description.

The script should be runnable as: `python prototypes/biobert_ner/run_ner.py`

It should handle the case where torch/transformers are not installed by printing a clear error message pointing to requirements.txt.

Create `prototypes/biobert_ner/README.md` with:
- What the prototype does
- Prerequisites (Python 3.11+, ~440MB model download on first run)
- Installation: `pip install -r prototypes/biobert_ner/requirements.txt`
- Usage: `python prototypes/biobert_ner/run_ner.py`
- Expected output files in results/
- Note: This is a prototype only, NOT part of the production pipeline
