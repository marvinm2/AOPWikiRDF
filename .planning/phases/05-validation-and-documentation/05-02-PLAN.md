  <action>
Create `.github/workflows/shacl-validation.yml` following the same pattern as `Turtle_File_Quality_Control.yml`:

1. **Triggers**: workflow_run (after "RDF Generation" completes with success) + workflow_dispatch (manual trigger). Use the exact workflow name from rdfgeneration.yml.
2. **Job**: runs-on ubuntu-latest, timeout-minutes 30
3. **Condition**: `if: github.event.workflow_run.conclusion == 'success' || github.event_name == 'workflow_dispatch'`
4. **Steps**:
   - Checkout repo (actions/checkout@v4)
   - Setup Python 3.11 (actions/setup-python@v4)
   - Install dependencies: `pip install rdflib pyshacl`
   - Run SHACL validation: `python scripts/run_shacl_validation.py`
   - Upload artifacts (actions/upload-artifact@v4, if: always()): shacl-report.ttl and shacl-summary.json
   - Update badge: Read status from shacl-summary.json, generate SVG badge to badges/shacl-validation.svg using a shields.io-style static badge (green for pass, red for fail). Use a simple Python script inline to generate the badge SVG (same pattern as uri-resolvability-check.yml if it generates badges, otherwise use a minimal SVG template).
   - Commit and push badge: Same git pattern as Turtle QC workflow (git config, git add badges/shacl-validation.svg, conditional commit+push)

Add `pyshacl` to `requirements.txt` (append to existing file, do not overwrite other entries).
