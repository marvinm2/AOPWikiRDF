Tests to create:

**1. test_pure_file_no_crossrefs:**
Load `data/AOPWikiRDF.ttl` with rdflib. Query for triples where:
- The predicate is owl:sameAs AND the subject starts with `cas:` or `pr:` namespace (these are the cross-ref patterns that should have moved to enriched).
- Specifically: count triples matching `?s owl:sameAs ?o` where `?s` is a chemical (cas:...) or biological object (pr:...) identifier.
Assert that no such cross-reference triples exist in the pure file.
NOTE: The pure file MAY still contain owl:sameAs in the "Mapped Gene identifiers" section (uniprot entries with rdfs:seeAlso/owl:sameAs to purl.uniprot.org) — that is expected per the research recommendation. The test should specifically check for CHEMICAL cross-references (objects in chebi:, chemspider:, wikidata:, chembl.compound:, pubchem.compound:, drugbank:, kegg.compound:, lipidmaps:, hmdb: namespaces) and PROTEIN ONTOLOGY cross-references (objects in pr: namespace from prodict).

**2. test_enriched_file_valid:**
Load `data/AOPWikiRDF-Enriched.ttl` with rdflib. Assert:
- File parses without error (valid Turtle)
- File has at least 100 triples (cross-references should be substantial)
- All predicates are either owl:sameAs or skos:exactMatch (no dc:title, dc:identifier, rdf:type etc.)

**3. test_enriched_file_header:**
Read `data/AOPWikiRDF-Enriched.ttl` as text. Assert:
- First line contains "AOPWikiRDF-Enriched.ttl"
- Header contains generation date reference or "Generated:"
- Header contains "Load alongside" or relationship text

**4. test_void_subsets:**
Load `data/AOPWikiRDF-Void.ttl` with rdflib. Assert:
- A triple `:AOPWikiRDF void:subset :AOPWikiRDF.ttl` exists
- A triple `:AOPWikiRDF void:subset :AOPWikiRDF-Genes.ttl` exists
- A triple `:AOPWikiRDF void:subset :AOPWikiRDF-Enriched.ttl` exists
Use SPARQL query or direct triple pattern match.

**5. test_void_enrichment:**
Load VoID file. Assert presence of:
- `dcterms:license <http://creativecommons.org/licenses/by/4.0/>` on parent or subsets
- `pav:importedFrom` on enriched subset referencing BridgeDb
- `void:triples` on at least one subset (non-zero integer)

**6. test_void_examples:**
Load VoID file. Assert `void:exampleResource` exists with at least 5 distinct objects (AOP, KE, KER, Chemical, Stressor).

**7. test_combined_triple_count:**
Load all three content files (AOPWikiRDF.ttl, AOPWikiRDF-Enriched.ttl, AOPWikiRDF-Genes.ttl) with rdflib. Sum their triple counts. This is a REGRESSION marker — store the sum and assert it is reasonable (> 150,000 total, based on current ~124,901 main + ~75,099 genes + cross-refs). This test documents the expected total rather than asserting an exact number since data changes weekly.

Mark all tests with `@pytest.mark.skipif(not os.path.exists("data/AOPWikiRDF.ttl"), reason="Requires generated data files")` to skip gracefully when data files are absent.

Import rdflib and use `Graph().parse(path, format='turtle')` for all file loading. Use standard rdflib namespace objects (OWL, VOID, DCTERMS, etc.) for triple pattern matching.</action>
