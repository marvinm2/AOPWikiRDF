**Part A: Create chemical_mapper.py**

Create `src/aopwiki_rdf/mapping/chemical_mapper.py` by extracting from pipeline.py lines 510-861 (the chemical mapping section). The primary function:

```python
def map_chemicals(chedict: dict, xml_root, aopxml_ns: str,
                  bridgedb_url: str, timeout: int = 30) -> dict:
    """Enrich chemical dict with BridgeDb cross-references.

    Args:
        chedict: Chemical dict from XML parser (keyed by chemical ID)
        xml_root: ElementTree root for iterating chemicals
        aopxml_ns: XML namespace string
        bridgedb_url: BridgeDb service URL
        timeout: Request timeout

    Returns dict with keys:
        'chedict': Enriched chemical dict with cross-reference properties
        'listofchebi': [chebi:XXXX, ...]
        'listofchemspider': [chemspider:XXXX, ...]
        'listofchembl': [chembl.compound:XXXX, ...]
        'listofdrugbank': [drugbank:XXXX, ...]
        'listofhmdb': [hmdb:XXXX, ...]
        'listofkegg': [kegg.compound:XXXX, ...]
        'listoflipidmaps': [lipidmaps:XXXX, ...]
        'listofpubchem': [pubchem.compound:XXXX, ...]
        'listofwikidata': [wikidata:XXXX, ...]
    """
```

The function must:
1. Iterate over chemicals in xml_root to extract CAS numbers (matching monolith lines 773-861)
2. Collect all CAS numbers from chedict entries
3. Call `batch_xrefs_chemical` from `mapping.bridgedb` (or implement inline if bridgedb.py not yet available -- use a try/except import)
4. Process batch results: for each chemical, add cross-reference properties to chedict entries using the cheminf codes (e.g., `cheminf:000407` for ChEBI)
5. Build the various listof* identifier lists
6. Return the result dict

IMPORTANT: The monolith iterates over XML `stressor` elements to find chemicals (lines 773-861), extracting CAS numbers from `stressor/chemicals/chemical/cas-number`. This iteration pattern must be preserved. The chemical dict gets enriched with properties like `cheminf:000407` (ChEBI), `cheminf:000405` (ChemSpider), etc.
