```python
def write_aop_rdf(filepath: str, entities: dict, prefix_csv_path: str,
                  clean_fn=None) -> None:
    """Write AOPWikiRDF.ttl from entity dictionaries.

    Args:
        filepath: Output file path (full path including filename)
        entities: Dict with keys: 'aopdict', 'kedict', 'kerdict', 'strdict',
                  'chedict', 'taxdict', 'celldict', 'organdict', 'bioobjdict',
                  'bioprodict', 'bioactdict', 'prodict', 'pro_hgnclist',
                  'pro_uniprotlist', 'pro_ncbigenelist'
                  Plus chemical identifier lists: 'listofchebi', etc.
        prefix_csv_path: Path to prefixes.csv
        clean_fn: Optional HTML tag cleaning function
    """
```
This is the largest section (~550 lines). It writes:
1. Prefixes from CSV (line 1297)
2. AOP triples (lines 1302-1500): iterate aopdict, write type, title, description, KEs, KERs, stressors, taxonomy, applicability
3. KE triples (lines 1501-1650): iterate kedict, write type, title, description, bio level, applicability, cell terms, organ terms, biological events, protein ontology links
4. KER triples (lines 1650-1750): iterate kerdict, write type, title, description, upstream/downstream KEs, weight of evidence
5. Stressor triples (lines 1750-1780): iterate strdict
6. Chemical triples (lines 1780-1830): iterate chedict, write cross-references

Copy the exact triple-building logic. Do not reformat or restructure the Turtle output -- it must match character-for-character for the regression test.
