```python
Wires extracted modules together in named stages with timing and logging.

logger = logging.getLogger(__name__)

def main(config: PipelineConfig | None = None) -> None:
    """Run the full AOP-Wiki XML to RDF conversion pipeline."""
    if config is None:
        config = PipelineConfig()
    # ... setup, stages ...
```

**Stage design** (per user decision: "Staged coordinator with named stages, logging, error handling, and timing per stage"):

```python
stages = [
    ("Setup & Static Files", _stage_setup),
    ("XML Download & Parse", _stage_parse),
    ("Chemical Mapping", _stage_chemicals),
    ("Protein Ontology Mapping", _stage_protein_ontology),
    ("HGNC Gene Mapping", _stage_gene_mapping),
    ("Write Main RDF", _stage_write_aop_rdf),
    ("Write Genes RDF", _stage_write_genes_rdf),
    ("Write VoID RDF", _stage_write_void_rdf),
]

context = {}
for stage_name, stage_fn in stages:
    t0 = time.time()
    logger.info(f"Stage: {stage_name} -- starting")
    stage_fn(config, context)
    elapsed = time.time() - t0
    logger.info(f"Stage: {stage_name} -- completed in {elapsed:.1f}s")
```

**Context dict data flow** (each stage reads/writes named keys):

1. `_stage_setup`: Resolve static files (reuse existing `_resolve_static_files`), set up logging. Writes `context['filepath']` = str(config.data_dir) + '/'.

2. `_stage_parse`: Call `parse_aopwiki_xml`. If config has aopwiki_xml_url, download XML first (the parser module handles download per user decision). Writes `context['entities']`, `context['xml_root']`, `context['aopxml_ns']`, `context['aopwikixmlfilename']`.

3. `_stage_chemicals`: Call `map_chemicals(chedict, xml_root, aopxml_ns, bridgedb_url, timeout)`. Reads entities from context. Writes `context['chemical_result']`.

4. `_stage_protein_ontology`: Build prolist from entities bioobjdict (matching monolith lines 1030-1048). Call `download_and_parse_promapping`. Writes `context['pro_result']`.

5. `_stage_gene_mapping`: Call `download_hgnc_data` -> `build_gene_dicts` -> `map_genes_in_entities` -> `build_gene_xrefs`. Reads entities, xml_root, aopxml_ns. Writes `context['gene_result']`.

6. `_stage_write_aop_rdf`: Assemble entities dict from context (merging entity dicts, chemical lists, protein ontology results). Call `write_aop_rdf`.

7. `_stage_write_genes_rdf`: Assemble gene_data dict from context. Call `write_genes_rdf`.

8. `_stage_write_void_rdf`: Assemble metadata dict from context (dates, filenames, modification times, BridgeDb info). Call `write_void_rdf`.

**CRITICAL ordering dependencies** (from RESEARCH.md Pitfall 4):
- Chemical mapping AFTER XML parsing (needs chedict, xml_root)
- Protein ontology AFTER XML parsing (needs bioobjdict for prolist)
- Gene mapping AFTER XML parsing AND HGNC download (needs kedict, kerdict, xml_root, genedict1/2)
- RDF writing AFTER all mapping stages

**CRITICAL variable shadowing** (from RESEARCH.md Pitfall 1):
- Protein ontology produces its own `hgnclist`, `uniprotlist`, `ncbigenelist` -- store as `pro_hgnclist` etc. in context
- Gene mapping produces its own `hgnclist` -- store as `gene_hgnclist` in context
- These are DIFFERENT lists used by DIFFERENT writer functions

Preserve the `main(config)` API signature. `run_conversion.py` calls `pipeline.main(config)` and must continue to work unchanged.
